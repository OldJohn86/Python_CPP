FPGA Images

FPGA #A:[Fa-xnlx-ddr.bit]
FPGA #B:[Ca53mp4.bit]

Build U-Boot

  1. Use G3 Openwrt build. U-boot will put at build_dir/target-aarch64-openwrt-linux-gnu/linux-g3_fpga/u-boot-2015.04/
  2. Or build u-boot stand alone: 
> git clone  gitolite@cs-local-gitmirror:sw/platforms/cs-unified/bootloaders/u-boot
> git checkout g3-v2015.04-master
> cd g3-v2015.04-master 
> make V=1 CROSS_COMPILE=aarch64-openwrt-linux- cortina_presidio-fpga_defconfig
> make V=1 CROSS_COMPILE=aarch64-openwrt-linux-

u-boot.bin and u-boot will generated at top directory of u-boot fold.

Build Kernel

Follow G3 Openwrt build procedure to generate kernel image.[1] Change directory to Linux folder and build device tree if .dtb is not generated. 

make ARCH=arm64 CROSS_COMPILE=aarch64-openwrt-linux- dtbs

ca7774-fpgaboard.dtb is located at build_dir/target-aarch64-openwrt-linux-gnu/linux-g3_fpga/linux-3.18/arch/arm64/boot/dts/cortina/ca7774-fpgaboard.dtb

JTAG Passowrd cmm

If JTAG password is enabled, you should run this cmm to assign JTAG passowrd before other cmm file. 

;TRST_N is asynchronous, but still asserts clocks
JTAG.PIN NTRST	0
JTAG.PIN TCK 	0
JTAG.PIN TCK 	1
JTAG.PIN TCK 	0
JTAG.PIN NTRST 	1
JTAG.PIN TCK 	0
JTAG.PIN TCK 	1
JTAG.PIN TCK 	0

;enter IR
JTAG.SHIFTTMS 0 0 0 0 1 1 0 0

;shift IR
JTAG.SHIFTREG 0 1 1 0

;enter DR
JTAG.SHIFTTMS 1 1 0 0

;shift DR, ID0, ID1
JTAG.SHIFTREG %l 0x0f85abcd 0x6537e22b   ; Default password
;JTAG.SHIFTREG %l 0x03020100 0x07060504

;shift TMS 0 to complete
JTAG.SHIFTTMS 1 1 0

JTAG OSC cmm

This cmm set OSC2/OSC3 bit on through JTAG interface. 

;TRST_N is asynchronous, but still asserts clocks
JTAG.PIN NTRST	0
JTAG.PIN TCK 	0
JTAG.PIN TCK 	1
JTAG.PIN TCK 	0
JTAG.PIN NTRST 	1
JTAG.PIN TCK 	0
JTAG.PIN TCK 	1
JTAG.PIN TCK 	0

;enter IR
JTAG.SHIFTTMS 0 0 0 0 1 1 0 0

;shift IR (lsb first)
JTAG.SHIFTREG 1 0 1 0

;enter DR
JTAG.SHIFTTMS 1 1 0 0

;shift DR, ID0, ID1
;JTAG.SHIFTREG %l 0x0f85abcd 0x6537e22b 
JTAG.SHIFTREG %Byte 0xf

;shift TMS 0 to complete
JTAG.SHIFTTMS 1 1 0

After FPGA bit file be download and board reset. Run this script(SP/MP2/MP4) from T32 tool. This script configure JTAG interface and hook debugger to target CPU(A53 MP2) and load u-boot binary to DDR offset 0x4000000. The "d.load.elf p:\space\Work\G3\g3-u-boot\u-boot /nocode /path "p:\"" specify the symbol path. p:\ is disk name on Windows system. On disk p:, the absolute path of symbol must be exist. Generally, you can create soft link which link to top of your path(e.g: ln -s /space).

Load Linux kernel image and trace into

After Uboot is running up, stop the processor on T32 and load kernel image with these command: 

d.load.binary P:\Work\G3_Master\G3-FPGA-150629\build_dir\target-aarch64-openwrt-linux-gnu\linux-g3_fpga\linux-3.18\arch\arm64\boot\Image 0x80000
d.load.binary P:\Work\G3_Master\G3-FPGA-150629\build_dir\target-aarch64-openwrt-linux-gnu\linux-g3_fpga\linux-3.18\arch\arm64\boot\dts\cortina\ca7774-fpgaboard.dtb 0x1000000
data.load.elf P:\Work\G3_Master\G3-FPGA-150629\build_dir\target-aarch64-openwrt-linux-gnu\linux-g3_fpga\linux-3.18\vmlinux /nocode /strippart "/space/Work/G3_Master/G3-FPGA-150629/build_dir/target-aarch64-openwrt-linux-gnu/linux-g3_fpga/" /path "p:\"
Break.set NSX:0x80228

My working directory is "/Work/G3_Master/G3-FPGA-150629/build_dir/target-aarch64-openwrt-linux-gnu/linux-g3_fpga/linux-3.18/". But the path is somehow too long for T32. So I strip some prefix path and create a soft link(linux-3.18) at p:\. So that T32 will search source code base on P:\linux-3.18. If you install T32 on Linux machine, it should be OK. Because there is no disk name on Linux OS, only absolute path is needed.

First three commands load kernel image to address 0x80000 and FDT image to 0x1000000. And also load kernel symbol. The last command set one break point right before switching to virtual space. Then continue the processor to run U-boot. And use command "booti 0x80000 - 0x1000000" to boot kernel. Uboot boot AARCH64 with command "booti" and it copy FDT content to another 8 byte align address before jump into kernel. Trace32 will stop at address 0x80228 which we set in T32. "br x27" will branch to address in register x27 which is the first instruction in virtual space. Just use function key F2 or step into, you will see the running space migrate from physical space to virtual space. And the next instruction will be located at start.S:__mmap_switched:

From now on, you can trace Linux kernel step by step. And dump memory or watch variable/structure inside Linux kernel.
