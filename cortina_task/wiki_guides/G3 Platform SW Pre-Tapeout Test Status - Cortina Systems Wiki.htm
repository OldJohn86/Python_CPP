OTP divisions

OTP is divided into the following 4 regions.

  • Region 1: Fixed location parameters
  • Region 2: Free available space, that is, BL1, not used when there is no hotfix code found in OTP
  • Region 3: NV counters
  • Region 4: OTP_BAD_BITMAP of 256 bits required for KP OTP

CHECHPOINTS

G3 Bootrom supports CHECKPOINTS that is a Cortina specific addition. BL1 supports upto a total 126 checkpoints. Only up-to 8 checkpoints can be activated. Two styles of checkpoints are available: regular checkpoint and replacement checkpoint.

G3 Bootrom defines 13 check-points 

typedef enum {
  CHECKPOINT_ID_UNDEF = 0,
  CHECKPOINT_ID_WATHDOG = 1,
  CHECKPOINT_ID_UART_INIT = 2,
  CHECKPOINT_ID_OTP_COPY = 3,
  CHECKPOINT_ID_NFLASH_INIT = 4,
  CHECKPOINT_ID_NFLASH_INIT_CONTINUE = 5,
  CHECKPOINT_ID_NFLASH_AFTER = 6,
  CHECKPOINT_ID_SFLASH_INIT = 7,
  CHECKPOINT_ID_SFLASH_INIT_CONTINUE = 8,
  CHECKPOINT_ID_SFLASH_AFTER = 9,
  CHECKPOINT_ID_MMC_INIT_BEFORE = 10,
  CHECKPOINT_ID_MMC_INIT = 11,
  CHECKPOINT_ID_MMC_AFTER = 12,
  CHECKPOINT_ID_BEFORE_BL2 = 13,
#if defined(TEST_OTP) || defined(TEST_ON_G3_ASIC_SIM)
  CHECKPOINT_ID_TEST1 = 124,
  CHECKPOINT_ID_TEST2 = 125,
  CHECKPOINT_ID_TEST3 = 126,
#endif
  CHECKPOINT_ID_INVALID = 127
} BL1_hotfix_checkpoint_id_t;

In case of a bootrom defect or to meet a new requirement which is not supported yet. We can deploy a hotfix on bootrom via a checkpoint to amend:

  • A generic code patch to amend the bootrom after otp_shadow copy;
  • A generic code patch to amend the bootrom before jumping to the next image;
  • Code patch on UART init;
  • Code patch on Watchdog init;
  • Code patch on OTP - after OTP copy;
  • Code patch on both serial and nand flash init before and after;
  • Code patch on SD/eMMC init before and after.

Comparison between G3 FPGA, G3 ASIC Simulator and G3 ASIC Chip

Considering the following differences between G3 FPGA, G3 ASIC Simulator and G3 ASIC chip, review the G3 bootrom (ASIC version) code to make sure functionalities work properly on ASIC even though these differences cannot be verified on FPGA or on G3 ASIC chip.

List of items to review:

  • Bus clock speed
    • G3 ARM A53 internal timer clock at 25MHz
    • G3 ASIC peripheral bus at 125 MHz while G3 FPGA peripheral bus at 25 MHz
    • BL1 udelay issue fixed
  • UART setting
    • UART setting to review by ASIC team
  • WDT setting
    • Make sure follow the spec, that is, 256 seconds
  • DDR controller initialization
    • Setup by BL2
    • Test not supported on FPGA and ASIC simulator before tape-out
  • PCIe/SATA/USB PHY
    • Setup not supported by BL1
  • OTP read and write
    • Not supported on FPGA. Tested via OTP_SIM software
    • Supported by G3 ASIC simulator
  • GLOBAL_STRAP
    • strap.sd supported on FPGA, not supported on ASIC simulator
    • strap.flash_pin supported on bothe FPGA and ASIC simulator
    • strap.speed not supported on FPGA
  • Bootup of CPUs should not power down (or de-active) anything even on OTP. Secondary CPU's in WFI is ok.

BL1 Test on ASIC simulator

On ASIC simulator, two types of bootrom are tested

  • Test-instrumented Bootrom with test instrumentation to monitor the test progress and the internal status;
  • Official ASIC bootrom release that is identical to the bootrom to be on G3 ASIC chip.

An official G3 bootrom is created by the script build-g3-asic-bootrom which enables CORTINA_G3_ASIC_OFFICIAL. An official G3 ASIC bootrom release consists of three files:

  1. g3-asic-bootrom-<timestamp>.txt - bootrom code in hex text format;
  2. g3-asic-bootrom-<timestamp>.md5sum - md5 check sum of the corresponding bootrom hex file;
  3. g3-asic-bootrom-<timestamp>.info - an information file containing records about how the release was created.

A test-instrumented bootrom is created by the following steps:

  • Enable TEST_ON_G3_ASIC_SIM with or without following additional build flags:
    • TEST_ON_G3_ASIC_SIM_HOTFIX
    • TEST_ON_G3_ASIC_SIM_OTP_WRITE
    • TEST_ON_G3_ASIC_SIM_OTP_SHADOW_COPY
    • TEST_ON_G3_ASIC_SIM_MULTIBOOT
  • Build an test-instrumented BL1 Bootrom for ASIC simulator 
make TEST_ON_G3_ASIC_SIM=1 MBEDTLS_DIR=../mbedtls TRUSTED_BOARD_BOOT=1 \
GENERATE_COT=1 \
ARM_ROTPK_LOCATION=regs \
ROT_KEY=../ca77xx-devel/ATF/TBB/development_keys/g3_rotprivk_rsa.pem \
V=1 PLAT=g3_fpga bl1
  • Translate bl1.bin into hex format to run on G3 ASIC simulator. Do: 
od -tx4 -w4 -Ax -v ./build/g3_fpga/release/bl1.bin | awk '{print $2}' | \
grep -v "^$" > test-on-hwsim.txt
